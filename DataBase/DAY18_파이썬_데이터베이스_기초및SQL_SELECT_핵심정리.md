# 파이썬_알고리즘_데이터베이스 기초 및 SQL SELECT 핵심 정리

### 1. 오늘 학습 요약 (핵심 정리)
- 데이터베이스의 기본 개념(구조, DBMS)과 관계형 데이터베이스(RDB)의 특징을 학습함
- 테이블 설계의 핵심인 키(Key)의 종류와 데이터 무결성, 정규화의 기초를 이해함
- SQL의 DML 중 `SELECT` 문을 활용한 다양한 데이터 필터링 및 조회 기법을 실습함

### 2. 배운 개념 정리
- **관계형 데이터베이스(RDB)**: 데이터를 행(Row)과 열(Column)로 구성된 테이블 형태로 관리하며, SQL을 사용해 데이터를 조작함
- **데이터베이스 관계**: 1:1, 1:N, M:N 관계가 있으며, M:N 관계는 '연관 테이블'을 통해 두 개의 1:N 관계로 분해하여 표현함
- **데이터 무결성**: 개체 무결성(PK), 참조 무결성(FK), 도메인 무결성을 통해 데이터의 정확성과 일관성을 유지함
- **비관계형 DB(NoSQL)**: Key-Value, Graph, Document, Wide-Column 등 데이터 특성에 맞는 다양한 저장 방식이 존재함

### 3. 오늘 작성한 코드 리뷰 
```sql
/* - SELECT: 조회할 컬럼 지정 (별칭 AS 활용)
   - FROM: 대상 테이블 지정 (별칭 c 활용)
   - WHERE: 필터링 조건 (BETWEEN, IN, LIKE 등 연산자 활용)
*/
SELECT c.name AS "국가명", c.population "인구"
FROM country c
WHERE c.continent = 'Asia'             -- 문자열은 작은따옴표 사용
  AND c.population > 1000000           -- 논리 연산자 AND 사용
  AND c.indepyear BETWEEN 1900 AND 2000 -- 범위 연산자 사용
  AND c.name LIKE 'K%';                -- 'K'로 시작하는 패턴 매칭
```
- **주의점**: `SELECT *` 보다는 필요한 컬럼만 명시하는 것이 성능상 유리하며, 테이블 별칭을 사용해 가독성을 높임

### 4. 실수/에러와 해결 과정
- **실수**: `WHERE` 절에서 조건식 뒤에 불필요한 컬럼명(`c.name`)을 나열하여 구문 에러 발생
- **해결**: `WHERE` 절은 필터링을 위한 '조건'만 명시하고 마무리해야 함을 인지하고 구문 수정

### 5. 실무/RAG 관점의 참견
- **RAG 관점**: AI가 자연어 질문을 SQL로 변환(Text-to-SQL)할 때, 테이블 간의 관계(FK)와 메타데이터(컬럼 설명)가 잘 정의되어 있어야 정확한 쿼리 생성이 가능함
- **설계 관점**: 실무에서는 데이터 중복을 방지하기 위해 제1정규형(1NF)부터 차근차근 정규화를 진행하여 데이터 이상현상을 방지하는 것이 필수적임
