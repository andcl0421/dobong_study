#  Instagram-style Database Architecture Design

본 설계는 이미지와 영상 등 미디어 콘텐츠에 최적화된 **인스타그램(Instagram) 서비스의 데이터베이스 구조**를 설계하고 최적화한 결과물입니다.

##  프로젝트 개요
- **목표**: 대용량 미디어 데이터 처리 및 소셜 상호작용(팔로우, 차단, 북마크)의 효율적 관리
- **핵심 기술**: 역정규화(Denormalization)를 통한 조회 성능 최적화, 다중 미디어 업로드 구조 설계

---

##  주요 설계 포인트 (Engineering Points)

### 1. 미디어 중심의 다중 업로드 구조 (`FeedImage`)
- 하나의 게시물(`Feed`)에 여러 장의 사진을 올릴 수 있도록 **1:N 관계**로 분리하여 유연한 미디어 관리가 가능하도록 설계했습니다.

### 2. 성능 최적화를 위한 역정규화 (`like_count`)
- 매번 수만 건의 좋아요 데이터를 `JOIN`하여 개수를 세는 방식은 성능 저하를 야기합니다.
- 이를 해결하기 위해 `Feed` 테이블에 **`like_count` 컬럼을 직접 관리**하여 조회 속도를 극대화하는 실무적 기법을 적용했습니다.

### 3. 소셜 관계의 독립성 (Follow & Block)
- **Follow**: 인스타그램 특유의 단방향 팔로우 로직을 구현했습니다.
- **Block**: 사용자 간의 차단 기능을 독립적인 테이블로 관리하여 데이터의 무결성을 확보했습니다.

### 4. 사용자 편의 기능 (`FeedBookmark`)
- 사용자가 저장하고 싶은 게시물을 따로 관리할 수 있는 북마크 기능을 설계하여 상호작용 범위를 넓혔습니다.

##  설계 리뷰 및 배운 점
- **비전공자/신입 개발자**로서 단순한 데이터 저장을 넘어 **'어떻게 하면 더 빠르게 데이터를 보여줄 수 있을까?'**에 대한 성능 최적화 고민을 실제 설계에 반영했습니다.
- `JOIN` 연산을 줄이기 위한 **역정규화**의 필요성과 그에 따른 데이터 정합성 유지의 중요성을 깊이 있게 이해하게 되었습니다.
