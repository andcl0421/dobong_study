## 🚀 실무 밀착형 요약: LLM Tool & Agent 루프 이해하기

오늘은 LLM(거대언어모델)이 단순히 글만 쓰는 단계를 넘어, **'도구(Tool)'**를 사용해 외부 세계와 소통하는 법을 배웠습니다. 

### 💡 핵심 개념: "LLM은 똑똑한 뇌, Tool은 손과 발"
LLM은 학습된 데이터 외에 최신 정보(오늘 날씨, 최신 영화 평점)는 알지 못합니다. 이때 **Tool**을 쥐여주면 LLM이 스스로 판단해서 "아, 이건 내가 모르니까 '날씨 검색기'를 써야겠다!"라고 결정합니다.

1.  **@tool 데코레이터**: 일반 파이썬 함수를 LLM이 읽을 수 있는 설명서로 변환합니다.
2.  **Tool 바인딩**: 모델에게 "너는 이제 이 도구들을 쓸 수 있어"라고 알려주는 과정입니다.
3.  **Agent 루프**: "질문 → 판단 → 도구 실행 → 결과 확인 → 최종 답변"으로 이어지는 반복 과정을 직접 코드로 구현해보았습니다.

---

## 🛠️ 실무 활용 (Business Logic & Design)

### 1. 설계 포인트: 왜 @tool의 설명(docstring)이 중요한가?
LLM은 코드를 실행하기 전, 함수의 이름을 보고 판단하는 게 아니라 **docstring(함수 설명)**을 읽고 언제 이 도구를 쓸지 결정합니다. 
- **나쁜 예**: `def func1(a): "데이터 가져오기"` (LLM이 언제 쓸지 모름)
- **좋은 예**: `def search_movies(query): "영화 제목으로 상세 정보를 검색한다. 예: 인셉션"` (LLM이 명확히 인지함)

### 2. 비즈니스 로직 Flow
사용자가 "인셉션 감독이 누구야?"라고 물으면 다음과 같은 로직이 작동합니다.
1.  **추론**: "인셉션"이 영화 제목임을 인지 → `search_movies` 호출 결정.
2.  **실행**: 영화 ID(`27205`)를 획득.
3.  **재판단**: 감독 정보를 알기 위해 ID를 들고 `get_movie_detail`을 다시 호출.
4.  **최종 응답**: 모든 정보를 조합해 사용자에게 자연어로 답변.

## 🔍 Troubleshooting: 오늘 겪은 에러와 해결

### Issue: Tool 실행 시 `KeyError` 또는 `AttributeError` 발생
- **원인**: LLM이 반환한 `tool_calls` 정보를 실제 함수와 연결해주는 `tool_map`이 없거나, `tool_call_id`를 맞춰주지 않아 발생했습니다.
- **해결**: 
  - `tool_map = {t.name: t for t in tools}`와 같이 딕셔너리를 만들어 이름을 기준으로 함수 객체를 즉시 찾을 수 있도록 매핑했습니다.
  - `ToolMessage`를 생성할 때 반드시 `tool_call_id`를 포함시켜 모델이 "어떤 질문에 대한 결과"인지 매칭할 수 있게 수정했습니다.

## 💻 실습 결과물 (Code Example)

````python
import os
import requests
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

# 1. 영화 상세 정보 조회 도구 정의
@tool
def get_movie_detail(movie_id: int) -> str:
    """
    영화의 ID(숫자)를 입력받아 감독, 출연진, 런타임 등 상세 정보를 반환합니다.
    반드시 search_movies를 통해 얻은 movie_id를 사용해야 합니다.
    """
    # 실무 팁: f-string 내에 변수명 오타 주의 (movie_Id -> movie_id)
    url = f"https://api.themoviedb.org/3/movie/{movie_id}"
    headers = {"Authorization": f"Bearer {os.getenv('TMDB_API_KEY')}"}
    params = {"language": "ko-KR", "append_to_response": "credits"}
    
    response = requests.get(url, headers=headers, params=params)
    data = response.json()
    
    # 감독(Director) 찾기 로직
    crew = data.get("credits", {}).get("crew", [])
    director = next((member['name'] for member in crew if member['job'] == 'Director'), "정보 없음")
    
    # 출연진(Cast) 상위 3명 추출
    cast = [member['name'] for member in data.get("credits", {}).get("cast", [])[:3]]
    
    return f"감독: {director} / 출연진: {', '.join(cast)} / 런타임: {data.get('runtime')}분"

# 2. Agent 실행부 구성
def run_movie_agent(user_query: str):
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
    tools = [search_movies, get_movie_detail] 
    tool_map = {t.name: t for t in tools}
    
    model_with_tools = llm.bind_tools(tools)
    messages = [HumanMessage(content=user_query)]
    
    while True:
        response = model_with_tools.invoke(messages)
        messages.append(response)
        
        # 더 이상 호출할 도구가 없으면 루프 종료
        if not response.tool_calls:
            break
            
        for tool_call in response.tool_calls:
            name = tool_call["name"]
            args = tool_call["args"]
            
            # 실제 도구 실행 (invoke)
            observation = tool_map[name].invoke(args)
            
            # 도구 실행 결과를 메시지 리스트에 추가
            messages.append(ToolMessage(content=str(observation), tool_call_id=tool_call["id"]))
            
    return response.content
