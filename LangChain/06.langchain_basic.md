## 🏗️ RAG 평가 전략 및 설계 포인트

### 1. Golden Dataset (신뢰의 기준) 구축
* **난이도별 구성**: 단일 문서 검색(Easy), 다중 조항 결합(Medium), 3개 이상의 문서 종합(Hard)으로 질문을 세분화하여 시스템의 한계를 명확히 파악했습니다.
* **사용자 관점 패러프레이즈**: "외부 저장 매체 연결 금지"라는 규정 용어 대신 "USB 꽂아도 돼?"와 같은 실제 사용자 말투로 질문을 구성하여 리트리버(Retriever)의 실전 성능을 테스트했습니다.

### 2. RAGAS 4대 핵심 지표 (RAG Triad + Recall)
* **Faithfulness**: 답변이 검색된 문서에만 근거하는가? (할루시네이션 방지)
* **Answer Relevancy**: 답변이 사용자의 질문 의도에 적절하게 대응하는가?
* **Context Precision**: 관련 있는 문서가 검색 결과 상단에 배치되었는가?
* **Context Recall**: 정답을 도출하기 위해 필요한 정보가 검색 결과에 누락되지 않았는가?

### 3. LLM-as-Judge (맞춤형 심사관) 구현
* RAGAS의 범용 지표 외에, **사내 규정 안내 챗봇**으로서 갖춰야 할 '정확성', '완전성', '적절한 톤'을 평가하는 전용 심사 로직을 추가했습니다.

## 🛠️ 핵심 구현 코드 (Evaluation Logic)

### [RAGAS 평가 자동화]
```python
from ragas import evaluate, EvaluationDataset, SingleTurnSample
from ragas.metrics import Faithfulness, ResponseRelevancy, LLMContextPrecisionWithReference, LLMContextRecall

# 1. 평가 메트릭 설정 (채점 선생님 임명)
metrics = [
    Faithfulness(llm=evaluator_llm),
    ResponseRelevancy(llm=evaluator_llm, embeddings=evaluator_embeddings),
    LLMContextPrecisionWithReference(llm=evaluator_llm),
    LLMContextRecall(llm=evaluator_llm)
]

# 2. 데이터셋 변환 및 평가 실행
samples = [SingleTurnSample(**r) for r in eval_results]
eval_dataset = EvaluationDataset(samples=samples)
result = evaluate(dataset=eval_dataset, metrics=metrics)

# 3. 수치 기반 성능 리포트 출력
df = result.to_pandas()
print(f"Average Faithfulness: {df['faithfulness'].mean():.4f}")
```

### [LLM-as-Judge 커스텀 평가]
```python
# Pydantic을 활용한 구조화된 평가 결과 도출
class JudgeResult(BaseModel):
    accuracy: int = Field(description="정확성: 문서 근거 여부 (1-5)")
    completeness: int = Field(description="완전성: 정보 누락 여부 (1-5)")
    tone: int = Field(description="톤: 사내 규정 안내에 적합한가 (1-5)")

judge_chain = judge_prompt | judge_llm.with_structured_output(JudgeResult)
```

## 🔍 트러블슈팅 (Troubleshooting)

### 이슈: 특정 난이도(Hard)에서 Context Recall 점수가 급격히 하락하는 현상
* **원인**: 여러 문서에 흩어진 정보를 종합해야 하는 질문인데, 검색 개수(`k=3`)가 너무 적어 필요한 정보가 유실됨.
* **해결**: 
    1. **k값 상향**: `search_kwargs={"k": 5}`로 늘려 정보 획득 확률 제고.
    2. **Multi-Query Retriever**: 사용자의 질문을 다양한 관점으로 재해석하여 검색 범위를 확장함.
* **결과**: Hard 난이도 질문에 대한 Context Recall 점수가 약 **25% 상승**함.
