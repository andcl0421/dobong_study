# 🚀 LangGraph 병렬 처리(Parallelization) 완벽 가이드

> **작성일:** 2026-04-03


에이전트가 복잡한 업무를 수행할 때, 모든 일을 하나씩 순서대로 하면 시간이 너무 오래 걸립니다. LangGraph의 **병렬 처리(Parallelization)**를 사용하면 여러 노드를 동시에 실행하여 전체 처리 속도를 획기적으로 줄일 수 있습니다.

---

## 🏗️ 비즈니스 로직 및 설계 핵심 포인트

### 1. 왜 병렬 처리가 필요한가요? (비전공자 가이드)
요리사가 혼자서 라면을 끓이고, 김밥을 말며, 튀김을 튀긴다고 가정해 봅시다. 
- **순차 처리:** 라면 끓이기(5분) → 김밥 말기(3분) → 튀김 하기(4분) = **총 12분**
- **병렬 처리:** 세 요리를 동시에 시작 = **총 5분 (가장 오래 걸리는 요리 시간)**

IT 실무에서도 법률 검토, 재무 분석, 기술 분석을 동시에 진행하면 사용자에게 훨씬 빠른 응답을 줄 수 있습니다.

### 2. 핵심 설계 패턴
* **Fan-out / Fan-in:** 한 지점에서 여러 갈래로 뻗어 나갔다가(Fan-out), 모든 작업이 끝나면 다시 하나로 모이는(Fan-in) 구조입니다.
* **Send API (동적 병렬):** 작업 개수가 미리 정해져 있지 않고, 실행 시점에 결정될 때 사용합니다. (예: 뉴스 기사가 3개면 3개, 10개면 10개 병렬 노드 생성)
* **Voting (투표):** 똑같은 질문을 여러 모델에게 던지고, 가장 좋은 답변을 골라내는 품질 향상 기법입니다.

## 🛠️ 실무형 트러블슈팅 (Troubleshooting)

### 이슈: 병렬 처리 시 데이터가 섞이거나 유실됨
* **현상:** 여러 노드가 동시에 결과를 반환할 때, `State`에 마지막 노드의 결과만 남고 이전 결과가 사라지는 현상.
* **원인:** 기본적으로 리스트 타입은 덮어쓰기 방식으로 동작하기 때문입니다.
* **해결:** `Annotated[list, operator.add]`를 사용하여 각 노드의 결과가 리스트에 **누적(append)**되도록 설정해야 합니다.

### 이슈: API 호출 한도 초과 (Rate Limit)
* **현상:** 너무 많은 노드를 동시에 돌리면 LLM API에서 `429 Too Many Requests` 에러를 뱉음.
* **해결:** `asyncio.Semaphore`를 사용하여 동시에 실행될 수 있는 노드 개수(예: 최대 3개)를 제한합니다.

## 💻 실무 표준 예시 코드

비전공자도 읽기 편하도록 변수명은 **Snake Case**를 적용하였으며, 결과의 순서를 보장하기 위해 인덱스를 포함하는 방식을 제안합니다.

```python
import operator
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 상태 정의 (결과를 누적하기 위해 operator.add 사용)
class AnalysisState(TypedDict):
    document_content: str
    analysis_results: Annotated[list, operator.add]
    final_summary: str

# 2. 분석 노드 (예: 법률 분석)
def analyze_legal_risks(state: AnalysisState):
    # 실제 로직에서는 LLM 호출이 들어갑니다.
    print("[법률] 분석 시작")
    return {"analysis_results": ["법률 리스크 없음"]}

# 3. 분석 노드 (예: 기술 분석)
def analyze_technical_points(state: AnalysisState):
    print("[기술] 분석 시작")
    return {"analysis_results": ["기술적 타당성 높음"]}

# 4. 그래프 구축
builder = StateGraph(AnalysisState)

builder.add_node("legal_node", analyze_legal_risks)
builder.add_node("tech_node", analyze_technical_points)
builder.add_node("summary_node", lambda state: {"final_summary": "종합 결과: 이상 없음"})

# 병렬 구조 연결 (START에서 두 노드로 동시 진입)
builder.add_edge(START, "legal_node")
builder.add_edge(START, "tech_node")

# 두 노드가 모두 끝나야 summary_node로 이동
builder.add_edge("legal_node", "summary_node")
builder.add_edge("tech_node", "summary_node")
builder.add_edge("summary_node", END)

graph = builder.compile()
```
