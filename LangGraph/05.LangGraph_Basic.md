# 🎯 LangGraph 마스터: 상황별 최적 패턴 선택 & 조합 가이드

> **작성일:** 2026-04-03


"모든 문제에 망치를 쓸 수는 없다." LLM 시스템 구축 시 발생하는 다양한 요구사항에 맞춰 어떤 '레고 블록(패턴)'을 조립할지 결정하는 기준을 정리합니다.

---

## 🔍 1. 문제 유형별 패턴 매칭 (Quick Guide)

내 프로젝트가 왜 '단순 LLM 호출'만으로 안 되는지 파악하면 답이 보입니다.

| 내 고민 (Problem) | 추천 패턴 (Pattern) |
|:---|:---|
| 어떤 도구(Tool)를 몇 번 써야 할지 감이 안 올 때 | **ReAct** (자율 판단 루프) |
| 질문 내용에 따라 상담 부서가 달라야 할 때 | **Router** (입력 분류 및 분기) |
| 답변 결과가 자꾸 틀리거나 품질이 들쭉날쭉할 때 | **Reflection** (자기 검토 및 수정) |
| "SEO 점수 80점 이상" 같은 명확한 채점 기준이 있을 때 | **Evaluator-Optimizer** (루브릭 기반 최적화) |
| 할 일이 너무 많고 순서대로 척척 진행해야 할 때 | **Plan-and-Execute** (계획 및 단계별 실행) |
| 5개의 웹사이트를 동시에 긁어와야 할 때 | **Parallelization** (병렬 처리) |
| 환불이나 결제처럼 실수하면 큰일 나는 작업이 있을 때 | **Human-in-the-Loop** (사람의 최종 승인) |

---

## 🏗️ 2. 실무에서 빛나는 '패턴 조합' 레시피

실제 서비스는 단일 패턴보다 **서브그래프(Subgraph)**를 활용한 조합형으로 설계됩니다.

* **Agentic RAG (RAG + ReAct):** 검색 결과가 없으면 검색어를 바꿔서 다시 찾거나, 다른 도구를 사용하는 동적 검색 시스템.
* **Safe Agent (ReAct + HITL):** 에이전트가 자유롭게 행동하되, `send_money` 같은 위험한 도구를 쓰기 직전에만 사람에게 '승인' 버튼을 띄우는 구조.
* **Smart Parallel (Router + Parallelization):** "A와 B를 비교해줘"라는 입력이 들어오면 Router가 이를 감지하고, A 분석 노드와 B 분석 노드를 동시에 실행한 뒤 합치는 방식.

## 📊 3. 트레이드오프: 성능 vs 비용 vs 속도

복잡한 패턴일수록 똑똑하지만, 그만큼 비싸고 느려집니다.

| 패턴 | 호출 횟수 | 비용(Cost) | 지연 시간(Latency) |
|:---|:---:|:---:|:---:|
| **단순 Chain** | 1회 | ⚪ (최저) | ⚡ (최단) |
| **Parallelization** | N회 | 🟡 (중간) | ⚡ (병렬이라 빠름) |
| **Reflection** | 2~6회 | 🟡 (중간) | 🐢 (순차 반복) |
| **Plan-and-Execute** | 5~15회 | 🔴 (높음) | 🐢🐢 (매우 김) |
| **Multi-Agent** | 10회+ | 🔴 (최고) | 🐢🐢🐢 (가장 김) |

> **멘토의 한마디:** 항상 **"가장 단순한 패턴(Simple Chain/RAG)"**으로 시작하세요. 거기서 한계를 느낄 때 비로소 다음 단계의 패턴을 도입하는 것이 실무자의 올바른 자세입니다.

## 📝 4. 실전 시나리오 해결 (연습 문제)

### [Case 1] AI 환불 매니저
* **요구사항:** 정책 문서 찾기 → 조건 검토 → 실제 환불 전 담당자 확인
* **정답 조합:** **RAG + ReAct + Human-in-the-Loop**
* **이유:** 지식 기반은 RAG로, 절차 수행은 ReAct로, 최종 안전장치는 HITL로 처리합니다.

### [Case 2] 전문 분야 상담사
* **요구사항:** 질문이 '일반/기술/영업'인지에 따라 다른 답변 제공
* **정답 패턴:** **Router**
* **이유:** 입력된 텍스트의 '의도(Intent)'를 먼저 파악하여 최적화된 경로로 보냅니다.

### [Case 3] SEO 블로그 에디터
* **요구사항:** 특정 키워드 포함 및 글자 수 등 엄격한 기준 충족
* **정답 패턴:** **Evaluator-Optimizer**
* **이유:** 단순 작성이 아니라, 정해진 '채점표'를 기준으로 통과할 때까지 다듬어야 하기 때문입니다.

## 💻 실무 표준 예시 코드 (Router + Parallelization)

```python
# 분석 대상을 병렬로 던지는 fan_out 노드 예시
def fan_out_analysis(state: State):
    """항목 리스트를 받아 개별 분석 노드('analyze')로 병렬 전송"""
    # Send(대상노드, 넘길데이터)를 리스트로 반환하면 LangGraph가 알아서 병렬 실행합니다.
    return [Send("analyze", {"item": name}) for name in state["items"]]

# 병렬 처리 결과를 하나로 합치는 리듀서(Reducer) 설정
class State(TypedDict):
    # operator.add를 써야 여러 노드에서 온 결과가 리스트에 차곡차곡 쌓입니다.
    analysis_results: Annotated[list[str], operator.add]
```
