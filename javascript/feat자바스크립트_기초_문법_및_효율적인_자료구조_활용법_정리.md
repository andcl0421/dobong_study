## 💡 비즈니스 로직과 설계 핵심 포인트

### 1. 자료형 선택의 중요성
실무에서는 단순히 값을 저장하는 것을 넘어, **"이 데이터가 변해도 되는가?"**를 먼저 고민해야 합니다. 
* 변경되지 않는 설정값은 `const`를, 반복문 등에서 변하는 값은 `let`을 사용합니다.
* `null`은 "의도적으로 비워둠"을, `undefined`는 "시스템적으로 값이 정의되지 않음"을 뜻하는 비즈니스 언어로 구분하여 사용합니다.

### 2. 엄격한 비교 (`===`) 권장
자바스크립트의 자동 형변환은 편리해 보이지만, 대규모 서비스에서는 치명적인 버그를 만듭니다. 
예를 들어, 사용자가 입력한 `"0"`(문자열)이 `false`로 취급되어 결제 로직이 꼬이는 것을 방지하기 위해 반드시 값과 타입을 모두 비교하는 `===`를 사용해야 합니다.

### 3. 자료구조의 목적별 활용
* **Array**: 순서가 있는 목록 관리 (예: 장바구니 리스트)
* **Object**: 속성이 정해진 개체 관리 (예: 회원 프로필)
* **Map**: 키값으로 다양한 타입을 써야 하거나 데이터 추가/삭제가 빈번할 때 (예: 실시간 유저 세션 관리)
* **Set**: 중복을 허용하지 않는 데이터 모음 (예: 오늘 방문한 유저 고유 ID 목록)

## 🛠️ 실무 예시 코드 (Clean Code)

````javascript
// 1. 회원 정보 관리 (Naming: user_data -> user_profile 권장)
const user_profile = {
    user_id: 1024,
    user_name: "홍길동",
    is_premium_member: true,
    login_count: 5
};

// 2. 프리미엄 회원 혜택 적용 (삼항 연산자 활용)
const discount_rate = user_profile.is_premium_member ? 0.15 : 0.05;
console.log(`${user_profile.user_name}님의 할인율은 ${discount_rate * 100}%입니다.`);

// 3. 중복 없는 장바구니 카테고리 추출 (Set 활용)
const cart_items = ["전자제품", "도서", "전자제품", "의류"];
const unique_categories = new Set(cart_items);

console.log("선택하신 카테고리 종류:");
unique_categories.forEach(category => {
    console.log(`- ${category}`);
});

// 4. 안전한 형변환 (명시적 형변환 사용)
const input_age = "25";
const numeric_age = Number(input_age); // 의도를 명확히 하여 버그 방지

⚠️ 트러블슈팅 (Troubleshooting)
1. typeof null이 object로 나오는 현상
문제: null은 객체가 아닌 원시 자료형인데 typeof 연산 시 object가 반환됨.

원인: 자바스크립트 초기 설계 오류이나 하위 호환성을 위해 유지됨.

해결: 데이터가 비어있는지 확인할 때는 typeof 대신 if (value === null)을 직접 사용하여 비교하는 것이 정확합니다.

2. 반복문 안의 스코프 문제
문제: for문 내부에서 선언한 변수를 외부에서 호출할 때 ReferenceError 발생.

원인: let, const로 선언된 변수는 블록 스코프({})를 가지기 때문임.

해결: 반복문 밖에서도 결과값이 필요하다면 블록 외부에서 let으로 변수를 선언한 뒤 내부에서 값을 갱신해야 합니다.

3. 문자열 연산의 함정 (+ vs -)
문제: "10" + 5는 "105"(문자열)가 되지만, "10" - 5는 5(숫자)가 됨.

원인: +는 문자열 연결을 우선시하고, -는 산술 연산을 우선시하여 자동 형변환이 일어남.

해결: 혼선을 막기 위해 항상 Number()를 사용하여 명시적으로 숫자로 바꾼 뒤 연산하는 습관이 중요합니다.
