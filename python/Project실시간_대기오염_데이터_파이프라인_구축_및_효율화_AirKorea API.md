# [Project] 실시간 대기오염 데이터 파이프라인 구축 및 효율화 (AirKorea API)

### 1. 프로젝트 목적 및 결과
- **목적**: 전국 17개 시도의 실시간 대기 정보를 수집하여 지역별 최적 환경 데이터를 추출하는 데이터 파이프라인 구현
- **결과**:
  - 서울/제주 지역 초미세먼지(PM2.5) 최저 농도 측정소 자동 선별
  - 종로구 등 특정 측정소 데이터에 대한 O(1) 접근이 가능한 자료구조 구축
  - 비동기 프로그래밍을 통한 데이터 수집 속도 개선

### 2. 핵심 학습 포인트 (Engineering Skills)
- **API 연동 최적화**: `requests`의 인코딩 이슈를 해결하기 위한 URL 쿼리 스트링 직접 제어 경험
- **비동기 처리(Asynchronous)**: `asyncio`, `aiohttp`를 활용하여 17개 시도 데이터를 병렬로 수집, 네트워크 대기 시간(I/O Bound) 최소화
- **데이터 예외 처리**: 측정 기기 점검 등으로 인한 결측치(`-`, `null`) 발생 시, `isdigit()`을 활용한 가드 클로즈(Guard Clause) 패턴 적용으로 프로그램 가용성 확보
- **자료구조 최적화**: 리스트 형태의 원본 데이터를 검색에 유리한 딕셔너리(`Hash Map`) 형태로 재구성

### 4. 기술적 의사결정 및 트러블슈팅
- **Issue**: API 키가 등록되었음에도 `JSONDecodeError` 발생
- **Cause**: 라이브러리의 자동 인코딩 기능이 공공데이터포털의 특수한 인증키 형식을 변형함
- **Solution**: 파라미터 전달 방식 대신 URL에 직접 키를 매핑하여 원본 데이터 보존 및 통신 성공
### 5. 실무 및 RAG 관점의 확장성
- **RAG(검색 증강 생성)**: 사용자가 특정 동네의 공기 질을 물었을 때, 본 프로젝트에서 구축한 딕셔너리 구조를 통해 실시간 컨텍스트를 LLM에 즉시 제공 가능
- **성능**: 17개 이상의 시도 데이터를 수집할 때 비동기 처리는 서버 리소스를 효율적으로 사용하며 응답 시간을 3배 이상 단축시킴
