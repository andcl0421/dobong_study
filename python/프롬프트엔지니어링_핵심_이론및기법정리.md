# [Deep Dive] LLM 성능 극대화를 위한 프롬프트 엔지니어링 전략

### 1. 프롬프트 엔지니어링의 정의
- LLM(대규모 언어 모델)이 사용자의 의도를 정확히 파악하고 최적의 결과물을 생성하도록 입력을 설계/최적화하는 기술.
- 단순한 '질문'을 넘어 문맥(Context), 지침(Instruction), 예시(Few-shot)를 체계적으로 구조화하는 과정.

### 2. 핵심 구성 요소 및 구조화 (Framework)
효과적인 프롬프트는 아래 4가지 요소가 유기적으로 결합되어야 함:
- **지시(Instruction)**: 모델이 수행할 구체적인 작업 (예: 요약, 분류, 추출)
- **문맥(Context)**: 모델의 응답 범위를 제한하는 배경 정보
- **입력 데이터(Input Data)**: 처리가 필요한 실제 원본 텍스트
- **출력 지시자(Output Indicator)**: 결과물 형식 지정 (예: JSON, Markdown Table)



### 3. 중급/고급 프롬프트 기법 (Advanced Techniques)
- **Few-shot Prompting**: 1개 이상의 입-출력 예시를 제공하여 모델이 패턴과 스타일을 즉시 학습하게 함.
- **CoT (Chain of Thought)**: "단계별로 생각하라"는 지시를 통해 복잡한 추론 과정의 논리적 오류를 방지.
- **XML Tagging**: `<context>`, `<instruction>` 등 태그를 사용해 프롬프트 각 섹션을 명확히 구분하여 모델의 파싱 정확도 향상.
- **Output Pre-filling**: AI의 응답 시작 부분(예: `{`)을 미리 작성하여 형식 오류를 강제적으로 방지.

### 4. 실무 관점의 최적화 전략
- **프롬프트 체이닝(Chaining)**: 복잡한 태스크를 단일 프롬프트로 처리하지 않고, 여러 하위 작업으로 분해하여 순차적으로 처리 (정확도 및 디버깅 효율 증대).
- **파라미터 제어**: Temperature(온도) 조절을 통해 결과의 창의성(High)과 결정론적 일관성(Low) 사이의 균형을 맞춤.
- **자체 교정(Self-Correction)**: 최종 답변 전 모델 스스로 계획을 검토하거나 논리적 오류를 체크하도록 지시하여 환각(Hallucination) 현상 억제.

### 5. 면접 대비: 왜 프롬프트 엔지니어링이 중요한가?
- AI 모델의 파라미터를 직접 수정(Fine-tuning)하지 않고도, 적은 비용과 시간으로 모델의 성능을 비약적으로 향상시킬 수 있는 경제적인 기술이기 때문임.
- 특히 RAG(검색 증강 생성) 시스템에서 검색된 지식(Context)을 모델에게 어떻게 전달하느냐에 따라 최종 답변의 질이 결정됨.
